このテンプレートは「**mitsuharu + Claude + Claude Code**」でのプロジェクト開発に最適化されています。
従来の要件定義書から不要な項目を削除し、AI との効率的な協働に必要な要素だけを厳選しました。

- **段階的記入**: 完璧を求めず、必要な部分から順番に埋める

## 🎯 Step 1: 方向性決め（AI との初回打ち合わせ）

### 1. プロジェクトの「なぜ」

**🤖 AI に相談：「この動機で開発する価値はある？他にも解決方法はない？」**

#### 作りたい理由

**【なぜこれを作ろうと思ったのか】**

1. **発信活動の基盤を作りたい**

   - 自作キーボード開発、デザイン、クリエイティブ活動を発信するメディアが必要
   - ZMK ファームウェア生成サービスや StreamDeck プラグインなど、複数のプロジェクトを紹介する場所

2. **収益化の柱の一つとして**

   - アフィリエイト(キーボード関連、ガジェット)
   - 広告収入
   - 企業からのデバイス提供(将来的に)
   - プロジェクトへの導線

3. **フリーランス復帰に向けた準備**

   - ポートフォリオとしての機能
   - 信頼性の構築
   - 継続的な情報発信によるブランディング

4. **AI 協働開発の実践**

   - Claude Code との協働開発を体験
   - 技術的知見の蓄積
   - Git 管理の習慣化

5. **WordPress 感を出したくない**
   - テンプレート感のあるブログは避けたい
   - 完全オリジナルデザインで差別化

#### 現在困っていること

**【今どんな課題があるのか】**

- 発信する場所が散らばっている(X、個人的なメモ等)
- プロジェクトの進捗を体系的に共有できていない
- ポートフォリオがない(実績を見せる場所がない)
- 記事ストックがないため、いざ発信しようと思っても準備に時間がかかる
- WordPress だと「The WordPress」感が出てしまう

#### 理想の状態

**【このシステムがあったらどうなっていたいか】**

- **継続的な情報発信**

  - 週 1〜2 回のペースで記事を投稿
  - ZMK プロジェクトの進捗、キーボードレビュー、ガジェット紹介など

- **収益化の実現**

  - アフィリエイトリンクからの収益
  - 月数千円〜数万円規模(初期目標)

- **ブランディング**

  - 「HaruStudio」としての統一感
  - プロフェッショナルな印象
  - 自作キーボードコミュニティでの認知

- **技術的成長**
  - Next.js、MDX、Vercel の運用スキル習得
  - Git 管理の習慣化
  - AI 協働開発のノウハウ蓄積

**✅ Claude の評価：**
発信活動とフリーランス準備を兼ねた、明確な目的がある。収益化の導線も考えられており、実現可能性が高い。

### 2. プロジェクトの「誰に」

**🤖 AI に相談：「このユーザー像で機能設計して大丈夫？想定漏れはない？」**

#### メインユーザー

- **どんな人？**:

  - 自作キーボードに興味がある人(初心者〜中級者)
  - ガジェット好き
  - 在宅ワーク環境を整えたい人
  - デザイン・クリエイティブ業界の人

- **どんな状況で使う？**:

  - PC で、仕事の合間や休日に情報収集
  - スマホで、通勤時間や隙間時間に読む
  - 新しいキーボードやガジェットを探している時

- **何に困ってる？**:

  - 自作キーボードの情報が散らばっている
  - 初心者向けの情報が少ない
  - どのキーボードを買えばいいかわからない
  - ファームウェアの書き方がわからない

- **何を期待してる？**:
  - わかりやすい解説
  - 実際の使用感レビュー
  - 初心者でも理解できる情報
  - 信頼できる情報源

#### 利用シーン（3 つまで）

1. **【キーボード選びの情報収集】**: 購入前に実際のレビューや比較記事を読む
2. **【プロジェクト進捗の追跡】**: ZMK ツールの開発状況を確認する
3. **【技術的な解決策を探す】**: ファームウェアのトラブルシューティング記事を検索

**【データベース】**

- 現時点では不要(MDX ファイルで管理)

**【ドメイン】**

- harustudio.info (ムームードメインで取得済み)

- **予算制約**:

  - ドメイン代: 年間 591 円(税込)(月約 49 円)
  - サーバー代: 0 円(Vercel 無料枠)
  - **合計: 月 49 円**

- **技術制約**:

  - mitsuharu は HTML/CSS の基礎知識あり
  - コード実装は Claude Code が担当

- **時間制約**:

  - 1 週間以内に初期版を完成させる

## 🎯 Step 3: 実装詳細（開発中に AI と段階的に詰める）

### 6. デザイン仕様の詳細化

#### 6.1. デザインコンセプト

**モダン × 温かみの融合**

**参考サイト:**

- Linear (https://linear.app/) - モダン、洗練、ダークモードの美しさ
- Josh Comeau's Blog (https://www.joshwcomeau.com/) - 温かみ、インタラクティブ、読みやすさ

**🚨 重要: 「AI 臭さ」「テンプレート感」の徹底排除**

最近、AI 生成デザインやテンプレート感のあるサイトが増えており、ユーザーは敏感に察知するようになっている。
以下の点に注意し、**細部までこだわった独自性のあるデザイン**を目指す:

**避けるべきパターン:**

- 均等なグリッドレイアウトに頼りすぎる
- よくあるカードデザイン(角丸 8px + シャドウ)をそのまま使用
- デフォルトの Tailwind カラーパレットをそのまま使用
- 汎用的なフォントサイズ比率(1.5 倍、2 倍等)
- 見たことあるアニメーション(フェードイン、スライドイン等の単調な使用)
- 「このサイト、どこかで見たことある」感

**目指すべき方向性:**

- 余白の取り方を独自に調整(16px, 24px ではなく、20px, 28px など)
- フォントサイズも微調整(16px → 16.5px, 17px など)
- ボタンやカードのスタイルをカスタマイズ(角丸の度合い、シャドウの強さ)
- 色の組み合わせを慎重に選定(単純な補色関係を避ける)
- アニメーションは控えめに、かつ独自性を
- **Linear や Josh Comeau を「参考」にするが、コピーしない**
- **技術的な実装面でも、見た目の細部でも、一つ一つ丁寧に作り込む**

**Claude Code への指示:**
デザインを実装する際は、「なんとなく良さそう」ではなく、**「なぜこの余白なのか」「なぜこの色なのか」を考えて実装**してください。
テンプレート的なコードをそのまま使わず、数値を微調整し、独自性を出すこと。

### 8. 技術的実装詳細

#### 8.1. MDX 処理フロー

**ライブラリ構成**

```json
{
  "dependencies": {
    "@next/mdx": "^14.x",
    "@mdx-js/loader": "^3.x",
    "@mdx-js/react": "^3.x",
    "gray-matter": "^4.x",
    "rehype-pretty-code": "^0.x",
    "rehype-slug": "^6.x",
    "remark-gfm": "^4.x"
  }
}
```

**処理の流れ**

1. `content/posts/` から MDX ファイルを取得
2. `gray-matter` でフロントマターをパース
3. `@next/mdx` で MDX を React コンポーネントに変換
4. `rehype-pretty-code` でシンタックスハイライト
5. カスタムコンポーネント(`MDXComponents.tsx`)を適用

#### 8.2. ダークモード実装

**使用ライブラリ**

```json
{
  "dependencies": {
    "next-themes": "^0.x"
  }
}
```

**実装方針**

- システム設定を優先
- デフォルトはダークモード
- localStorage に設定を保存
- トグルボタンで切り替え

#### 8.3. 目次(TOC)生成

**実装方法**

- MDX の見出し(`h2`, `h3`)を抽出
- `rehype-slug` で ID を自動付与
- スクロール位置に応じてアクティブ表示
